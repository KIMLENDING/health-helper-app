# 컴포넌트 재사용성 개선 계획

현재 `health-helper-app`의 컴포넌트들을 분석한 결과, 재사용성을 크게 개선할 수 있는 영역을 발견했습니다.  
주요 패턴으로 **제네릭 타입화**, **Compound Component 패턴**, **공통 로직 추출**을 적용할 예정입니다.

---

## 개선 영역 요약

| 영역 | 현재 문제 | 적용 패턴 |
|------|-----------|-----------|
| 차트 컴포넌트 4개 | `data: any` 타입, 중복 구조 | Generic + BaseChart Compound Component |
| `CardContainer` | `className` 커스터마이징 불가 | Props 확장 (className, variant) |
| `DataTable` | 도메인 로직 하드코딩 (운동 목록 고정) | Generic TData화 + 외부 주입 |
| 로딩/에러 상태 | `showWeek`, `showPlans`, `showExercisePlan` 각각 구현 | `AsyncStateWrapper` 공통 컴포넌트 |
| `showWeek` 내 요일 버튼 | JSX 조건 분기가 130줄 이상 중복 | `DayCell` 서브컴포넌트 분리 |

## 제안하는 변경 사항

### 1. 차트 컴포넌트 — Compound Component + Generic 타입

#### 현재 문제
- `TimeByDay`, `WeightByDay`, `TitlebyDay`, `WeightByPart` 4개 컴포넌트 모두 `{ data: any }` 타입 사용
- `ChartContainer`, `CartesianGrid`, `XAxis`, `ChartTooltip`, `ChartLegend` 구조가 거의 동일하게 반복됨
- `WeightByPart`는 자체 `chartConfig`를 별도로 선언 (다른 3개는 `lib/utils`에서 import)

#### 개선 방향: `BaseChart` Compound Component
```tsx
// [NEW] components/UserCpmponents/chartComponents/BaseChart.tsx
<BaseChart config={chartConfig} data={processedData}>
  <BaseChart.Grid />
  <BaseChart.XAxis dataKey="day" />
  <BaseChart.Tooltip formatter={customFormatter} />
  <BaseChart.Legend />
  <BaseChart.Bar dataKey="totalTime" />   {/* 또는 <BaseChart.Line /> */}
</BaseChart>
```

**효과**: 
- 각 차트는 데이터 가공 로직만 가지고, 렌더링 구조는 `BaseChart`가 담당
- 새로운 차트 추가 시 `BaseChart`의 서브컴포넌트 조합만으로 구성 가능

### 2. `DataTable` — 도메인 의존성 분리

#### 현재 문제
```tsx
// 현재: 운동 목록 전용 로직이 컴포넌트 내부에 하드코딩
const availableTags = ["상체", "하체", "가슴", "등", "어깨", "팔", "허벅지", "종아리"]
// 헤더 타이틀도 "운동 목록" 고정
<h2>운동 목록</h2>
// useSelectedExercise 뮤테이션도 내부에서 직접 호출
const useSelectedExerciseMutation = useSelectedExercise();
```

#### 개선 방향: Props로 외부 주입

```tsx
interface DataTableProps<TData, TValue> {
  columns: ColumnDef<TData, TValue>[]
  data: TData[]
  // 추가
  title?: string             // "운동 목록" → 외부에서 주입
  titleIcon?: React.ReactNode
  filterKey?: string         // "title" → 외부에서 주입
  availableTags?: string[]   // 태그 목록 외부 주입
  onRowSelectionChange?: (rows: TData[]) => void  // mutation 분리
}
```

**효과**: 운동 목록 외에 다른 데이터(회원 목록, 세션 기록 등)에도 재사용 가능

### 4. 로딩/에러 상태 — `AsyncStateWrapper` 공통 컴포넌트

#### 현재 문제
`showWeek`, `showPlans`, `showExercisePlan` 3개 컴포넌트 각각에 로딩 스켈레톤 + 에러 UI가 중복 구현됨

```tsx
// showPlans.tsx
{isLoading ? (
  <div className="flex flex-col gap-3">
    {[1,2,3].map(i => <div key={i} className="h-20 animate-pulse ..."/>)}
  </div>
) : isError ? (
  <div>데이터를 불러오는 중 오류가 발생했습니다...</div>
) : data ? ( ... ) : null}

// showExercisePlan.tsx — 동일한 패턴 반복
// showWeek.tsx — 동일한 패턴 반복
```

#### 개선 방향: Generic `AsyncStateWrapper<T>`
```tsx
// [NEW] components/LayoutCompents/AsyncStateWrapper.tsx
interface AsyncStateWrapperProps<T> {
  isLoading: boolean;
  isError: boolean;
  data: T | undefined;
  loadingSkeleton?: React.ReactNode;   // 커스텀 스켈레톤
  errorMessage?: string;
  emptyState?: React.ReactNode;        // 데이터 없을 때 UI
  children: (data: T) => React.ReactNode;  // 제네릭 렌더 함수
}

// 사용
<AsyncStateWrapper
  isLoading={isLoading}
  isError={isError}
  data={data}
  loadingSkeleton={<PlanSkeleton />}
  emptyState={<EmptyPlanState />}
>
  {(plans) => plans.map(plan => <PlanCard key={plan._id} plan={plan} />)}
</AsyncStateWrapper>
```

### 5. `showWeek` — `DayCell` 서브컴포넌트 분리

#### 현재 문제
`showWeek.tsx`의 140~180번 라인에 걸쳐 `day.id` 유무에 따른 조건 렌더링이 중복 (클래스명만 약간 다름)

#### 개선 방향
```tsx
// showWeek.tsx 내부에 로컬 서브컴포넌트로 분리
const DayCell = ({ day }: { day: WeekDay }) => {
  const circleClass = cn(
    "w-10 h-10 sm:w-14 sm:h-14 rounded-full flex items-center justify-center transition-all duration-300",
    day.status && "bg-green-500 text-white shadow-md hover:bg-green-600",
    !day.status && "bg-gray-100 dark:bg-gray-800 text-gray-700 dark:text-gray-300",
    day.isToday && "border-2 border-blue-500 dark:border-blue-400"
  );
  const content = <div className={circleClass}>...</div>;
  return day.id ? <Link href={...}>{content}</Link> : content;
};
```

## 우선순위

> [!IMPORTANT]
> 아래 순서대로 진행할 것을 권장합니다. 1~3번은 독립적으로 적용 가능하고, 4~5번은 3번 이후에 진행하는 것이 자연스럽습니다.

| 순위 | 항목 | 효과 | 난이도 |
|------|------|------|--------|
| 🥇 1 | `AsyncStateWrapper` 추가 | 코드 중복 즉시 제거, 3개 컴포넌트에 적용 | 낮음 |
| 🥈 2 | `CardContainer` Props 확장 | 변경량이 적고 즉시 효과 큼 | 낮음 |
| 🥉 3 | `showWeek` → `DayCell` 분리 | 가독성 크게 향상 | 낮음 |
| 4 | `DataTable` 도메인 분리 | 재사용성 최대화 | 중간 |
| 5 | `BaseChart` Compound Component | 차트 확장성 확보 | 높음 |

## 검증 계획

- 각 항목 적용 후 기존 동작이 동일한지 브라우저에서 확인
- TypeScript 타입 에러 없는지 `tsc --noEmit`으로 확인
- `data: any` 로 남아있는 부분 최소화 여부 검토
